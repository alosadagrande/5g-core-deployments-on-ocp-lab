= Check MNO Deployment is Finished
include::_attributes.adoc[]
:profile: 5g-ran-lab

In the previous section we have learned how to follow the cluster deployment process, but the cluster finished its deployment doesn't mean that the OpenShift cluster deployment is complete.

We say that the cluster deployment finished when the cluster has been deployed *and* day2 configurations have been applied. In this section we will learn how to verify that the configs have been applied and our cluster is ready to run 5G Core workloads.

[#check-mno-deployment-webui]
== Check MNO Deployment has Finished via the WebUI

1. Access the https://console-openshift-console.apps.hub.5g-deployment.lab/multicloud/home/welcome[RHACM WebUI] and login with the OpenShift credentials.
2. Once you're in, click on `Infrastructure` -> `Clusters`. You will see a screen like the one below, notice that there is a label saying `ztp-done` for the MNO cluster (that means ztp pipeline has finished):
+
image::acm-sno-abi-ztp-done.png[RHACM SNO ABI ztp-done]
+
3. Additionally, we can verify that all policies are compliant for our `mno` cluster. On the RHACM WebUI go to `Governance` -> `Policies` and you will see the following screen where we can see all policies are green:
+
IMPORTANT: Applying policies on `mno` can take up to 15 minutes after cluster has been fully deployed.
+
4. At this point the `mno` cluster is ready to run 5G Core workloads.

[#check-mno-deployment-cli]
== Check MNO Deployment has Finished via the CLI

1. Check the `ManagedCluster` for MNO cluster (including its labels). You should see the `ztp-done` label, just as we saw in the WebUI.
+
[.console-input]
[source,bash,subs="attributes+,+macros"]
-----
oc --kubeconfig ~/hub-kubeconfig get managedcluster mno --show-labels
-----
+
[console-input]
[source,console]
-----
NAME   HUB ACCEPTED   MANAGED CLUSTER URLS                         JOINED   AVAILABLE   AGE    LABELS
mon    true           https://api.sno-abi.5g-deployment.lab:6443   True     True        122m   <OMITTED_LABELS>,ztp-done=
-----
+
2. Check the policies are compliant for our MNO cluster.
+
[.console-input]
[source,bash,subs="attributes+,+macros"]
-----
oc --kubeconfig ~/hub-kubeconfig -n mno get policies
-----
+
IMPORTANT: Applying policies on `mno` can take up to 15 minutes after cluster has been fully deployed.
+
[console-input]
[source,console]
-----
NAME                                        REMEDIATION ACTION   COMPLIANCE STATE   AGE
ztp-policies.common-config-policies         inform               Compliant          74m
ztp-policies.common-subscription-policies   inform               Compliant          74m
ztp-policies.du-sno-group-policies          inform               Compliant          74m
ztp-policies.du-sno-sites-sites-policy      inform               Compliant          74m
-----

3. At this stage `mno` OpenShift cluster is ready to run 5G Core workloads.
